SAE 103

  Objectifs

  1. Administrer des systèmes informatiques communicants complexes.
  2. Installer et configurer une machine.
  3. Créer une notice d’utilisation.

  Sujet

L'objectif de cette SAE était de configurer une machine virtuelle selon un cahier des charges.

De plus, une notice devra être rédigée, cette notice devra expliquer comment la machine a été configurée.

Il est aussi demandé de proposer une configuration matérielle pour un poste destine au développement web.

Ce poste comprend une machine fixe, un ordinateur portal et une station d'accueil pour l'ordinateur portable.

À droite, vous pourrez voir une capture d'écran de virtual box.

  Livrables et Documents

Pour cette sae nous avions plusieurs livrables :

  • Un dossier d'étude et de choix des solutions, concernant le poste informatique. Ce dossier comprend :
    1. Une courte présentation de la demande formulée par le client et de la façon dont vous comptez répondre dans les parties qui suivent
    2. Description des architectures matérielles proposées
    3. Préconisation d'un poste de développement (architecture matérielle et architecture logicielle préconisées) en justifiant le choix

  • Une notice d'installation de la machine virtuelle, qui comprend une explication de comment est faite la configuration :
    1. du système de base
    2. en mode texte
    3. de l'environnement de bureau
    4. des outils divers
    5. des outils pour le développement web

  • Machine installée et configurée
    • Le fichier OVA faisant plusieurs GO, il est compliqué de le mettre à disposition, il n'est donc pas disponible dans le répository

  • Une soutenance ou non devions présenter notre machine virtuelle

  Mon travail

Lors de ce travail, j'ai pris le rôle de chef de projet.
J'ai donc dû gérer l'équipe avec laquelle je travaillais, et j'ai donc réparti les tâches. Je me suis vite rendu compte que mes camarades de travail ne réalisaient pas leur part de travail et que je devrais donc faire leur par en plus de la mienne.
Pour qu'ils fassent leurs pars dans le projet. J'ai donc du mettre en place plusieurs mesure pour qu'ils fassent leur pars dans le projet :
  • Des créneaux où je leur proposais de leur expliquer comment le code fonctionnait et comment faire leurs parties.
  • Un system de pénalité ou ceux que ne rendait pas leur travail en temps et en heure risque de se faire radier du groupe.
Ce système n'étant pas en place lors du début du projet, j'ai donc dû réaliser les rapports ainsi que la majeur partie de la configuration de la machine virtuelle.

  Compétence Obtenue

Lors de ce travail, j'ai obtenu les compétences suivantes :
  1. Gestion d'une équipe qui ne travaille pas.
  2. Rédaction de rapport sur différents sujets.
  3. Configuration d'un poste informatique.
  4. Conception d'un configuration hardware.
