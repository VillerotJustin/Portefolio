SAE 1.03 Installation of a Computer workstation
Objective of the project
  • Administer complex communicating IT systems.
  • Install and configure a machine.
  • Create user manuals.
Subject
The aim of this SAE was to configure a virtual machine according to specifications.
In addition, a manual had to be written, explaining how the machine was configured.
We were also asked to propose a hardware configuration for a workstation intended for web development.
This workstation comprises a fixed machine, a portal computer and a docking station for the laptop.
On the right, you can see a screenshot of the virtual box.
Deliverables & Documents
For this sae we had several deliverables:
  • A file for studying and selecting solutions for the computer workstation, including:
    • A short presentation of the customer's request and how you intend to respond in the following sections
    • Description of proposed hardware architectures
    • Recommendation of a development workstation (recommended hardware architecture and software architecture), justifying the choice.

  • Virtual machine installation instructions, including an explanation of how to configure the virtual machine:
    • of the basic system
    • in text mode
    • desktop environment
    • various tools
    • web development tools

  • Machine installed and configured (As the OVA file is several GB in size, it is complicated to make it available, so it is not available in the repository.)

  • A presentation where we had to present our virtual machine
My Work
During this job, I took on the role of project manager.
This meant I had to manage the team I was working with, so I divided up the tasks.
I soon realized that my workmates weren't doing their share of the work, so I'd have to do theirs as well as mine.
To get them to play their part in the project. So I had to implement several measures to get them involved in the project:
  • Time periods where I offered to explain to them how the code worked and how to do their parts.
  • A penalty system where those who didn't hand in their work on time risked being removed from the group.
As this system was not in place at the start of the project, I had to create the reports and do most of the virtual machine configuration.
Skill Obtained
During this project, I obtained the following skills:
  • Management of a team with difficulties.
  • Writing reports on various subjects.
  • Configuring a computer workstation.
  • Hardware configuration design.
Links
  • Github Project.

SAE 1.04 Database creation
Objective of the project
  • Design a database based on a customer's requirements.
  • Implement a web interface to interact with a database.
Subject
The aim of this SAE was to create a web application that responded to the topic we were given.
In our case, we had to create a web application enabling an association renting bicycles to manage :
  • their bikes
  • customers
  • loans
What's more, the application had to be designed using the flask framework.
Deliverables & Documents
For this sae we had several deliverables:
  • Requirements analysis report
  • Data inventory and conceptual data model
  • Database creation script.
  • Dataset
  • Web interfaces (server in Python) for displaying/reading/adding/modifying/deleting data in tables.
  • Queries with aggregation functions to extract data and display results in the browser.
You can find these deliverables in the GitHub repository.
My Work
During this job, I took on the role of project manager.
This meant I had to manage the team I was working with, so I divided up the tasks.
I soon realized that my workmates weren't doing their share of the work, so I'd have to do theirs as well as mine.
So that they could do their part in the project. So I had to implement several measures to get them to do their part in the project:
  • Periods where I offered to explain how the code worked and how to make their part. (In particular, I offered a weekday afternoon slot before the personal games were due, where I explained to the members who came how their game worked and how they could implement it.)
  • A penalty system for those who don't hand in their work on time, with the risk being struck off the group.
This system was set up after the implementation of the web interface for displaying/reading/adding/modifying/deleting data in tables.
I therefore had to implement this part, which was a group note.
Skill Obtained
During this project, I obtained the following skills:
  • Managing a difficult team.
  • Study a company's needs.
  • Designing a product to meet needs.
  • Development of a web application managing a database.

SAE 1.02 Needs assessment
Objective of the project
  • Formalize a customer's expectations for a given project.
  • Familiarize yourself with project management based on customer needs.
Subject
The subject of the survey was to choose an event to plan and explain how the group organized itself to plan the event. The topic chosen by the donations group I was part of was : - Organize an event for the opening of a video and board game store.
Deliverables & Documents
For this sae we had several deliverables:
  • questionnaires and provisional schedule
  • the slideshow for the presentation
As this was the first sae we did during our training course, using GitHub was not an option. This made it difficult to make the deliverables available.
My Work
For this assignment, I took on the role of a group member.
I had to research what we could organize for the store opening.
As the store was a video and board game store, we decided to highlight this in a number of ways:
  • Stand where you can try out some board games and video games
  • Invitation from a regional board game designer
  • Organization of a tournament on arcade machines
During this project, the fact that everyone did a bit of everything made it difficult to distinguish between personal and group work.
On the other hand, I can say that I was responsible for researching and budgeting the tournament prizes, as well as the sound and light equipment.
Skill Obtained
During this project, I obtained the following skills:
  • Manage a project as part of a team.
  • Report on project progress.
  • Present a project orally.
